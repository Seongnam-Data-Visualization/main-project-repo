# 성남시 데이터 활용 공모전_PLAN

1. 배경(상황파악)
    
    성남시가 지금 2035 도시기본계획을 실행 중. 기존의 수정·중원·분당 3중심 체계를 버리고, 성남 전체를 북부/중부/남부 생활권으로 재편한다는 내용이 핵심적이다. 즉, 행정이 지금 생활권 경계를 다시 긋고 있는 시점이다.
    
2. 문제인식
    
    두가지 결함 체크
    
    1. 도시개발계획 자료집 데이터 분석의 부정확성
        
         "수정구에 사회복지시설 134개"처럼 단순 현황 나열 수준에 머물러 있다. "수정구의 고령인구 대비 복지시설이 분당구보다 얼마나 부족한가"와 같은 수요-공급 분석이 없다는게 허점.
        
    2. 행정이 그은 생활권 경계가 시민의 실제 생활과 일치하는지에 대한 검증이 없다.
    "북부 생활권으로 묶으면 된다"고 계획했는데, 북부 시민들이 실제로 분당으로 소비하러 가고 있다면 그 계획은 현실과 동떨어진 것일 수 있기에 이 부분을 실제 데이터 분석을 통해 살펴봐야 한다. 

1. 궁극적 목표(이 데이터 분석을 왜 하는가)
    
    성남시가 현재 추진 중인 생활권 재편 계획이 잘못된 방향이라는 걸 데이터로 증명하고, 더 나은 대안을 제시한다??
    
    | 질문 | 분석으로 답하는 것 |
    | --- | --- |
    | 지금 계획이 왜 부족한가? | 행정 생활권 경계 ≠ 시민 체감 이동 경계임을 정량화 |
    | 어디가 진짜 문제인가? | 수요↑ 공급↓ 접근성↓ 가 겹치는 지역을 Type D로 특정 |
    | 어떻게 바꿔야 하는가? | **Type별로 다른 처방** (연결 보완 / 시설 추가 / 경계 조정) 제안 |

[**#3 체감이동 GAP Score** ](%EC%84%B1%EB%82%A8%EC%8B%9C%20%EB%8D%B0%EC%9D%B4%ED%84%B0%20%ED%99%9C%EC%9A%A9%20%EA%B3%B5%EB%AA%A8%EC%A0%84_PLAN/#3%20%EC%B2%B4%EA%B0%90%EC%9D%B4%EB%8F%99%20GAP%20Score%2035a993e510ff8068aa4ad45cfe1ab3fa.md)

각각의 동들을 세가지 score로 계산해서 합산하여, 각 행정동에 관한 최종 score을 낸다. 

그리고 Type A, Type B, Type C, Type D 로 유형을 나누어서 각 행정동이 어떤 타입인지 분류한다.    

- Type 에 대한 분류
    
    세 Score 조합으로 타입이 결정된다.
    
    Type D가 최악의 조합인 것…. 세가지 부문의 score 다 높은 동 = Type D (복합 취약형) → 최우선 개선 대상
    
    | 타입 | 공급 GAP | 접근성 | 체감이동 GAP | 의미 | 해결책 | 예상 지역 |
    | --- | --- | --- | --- | --- | --- | --- |
    | **A** | 낮음  | 양호  | 낮음  | 시설 충분, 이동 편리, 주민이 생활권 안에서 자급자족 | 현상 유지 | 분당·판교 일부 |
    | **B** | 높음  | 보통  | 높음  | 시설이 부족해서 주민이 타 생활권으로 빠져나감. 관외소비·이동 비율 높음 | 시설 추가 공급 | 위례·수정구 일부 |
    | **C** | 낮음  | 불량  | 보통 | 시설은 있는데 교통 연결이 안 돼서 실제로 이용 못 함 | 교통 연결 보완 | 중원구 구시가지 |
    | **D** | 높음  | 불량  | 높음  | 시설도 부족하고, 접근도 어렵고, 결국 외부로 이동. 세 문제가 동시에 발생 | 시설 + 교통 + 경계 재조정 **최우선** | 수정구 북부 |
    
    ---
    

공급 GAP Score    → "시설이 부족한가"                 
접근성 Score      → "시설이 있어도 못 가는가"         
체감이동 GAP Score → "결국 어디로 가는가"
