# #2 접근성 score

시설이 있어도 거기까지 갈 수 있는지를 나타내는 점수.

| 데이터 | 출처 | 사용 속성 |
| --- | --- | --- |
| 복지시설 위치 좌표 |  |  |
| 버스 정류장 위치 + 노선 |  |  |
| 지하철 역사 위치 |  |  |
| 도로 네트워크 GIS |  |  |

### 분석법

1. 물리적 접근성
    
    각 복지시설로부터 반경 300m(도보 약 4분)이내 버스정류장 수 집계 → 행정동별 평균값으로 집계
    
2. 실제 이동 가능성(환승, 이동 소요시간 등 단순한 거리값만 갖고 산출하는 것을 넘어서 실제로 사람들이 접근할 수 있는지) ⇒ *이 부분 구현이 좀 까다로울 수도..*
    
    출발지 : 행정동 중심점 (또는 거주 밀집 지점)
    도착지 : 복지시설
    측정값 : 대중교통 환승 횟수 / 이동 소요 시간
    도구 : OSMnx + GTFS 데이터 or 카카오맵 API
    
3. inter vs intra 생활권 비교
    
    [예시]
    
    수정구 A동 주민이 수정구 내 시설 이용 시 : 이동시간 X분
    수정구 A동 주민이 분당구 시설 이용 시   : 이동시간 Y분
    
    Y < X 이면 → 행정 생활권 경계가 실제 이동 패턴과 불일치 ⇒ 생활권 경계가 잘못 그어졌다는 증거가 됨.
    

### 접근성 score 산출법

```
접근성 Score(행정동) =
  w₁ × 시설 반경 내 정류장 수 (정규화)
+ w₂ × 평균 환승 횟수 (역수, 낮을수록 좋음)
+ w₃ × 평균 이동 소요시간 (역수)
```
