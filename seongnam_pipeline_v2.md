## 분석 개요

### 배경 : 성남시가 지금 생활권 경계를 다시 긋고 있다

성남시는 **2035 도시기본계획**을 통해 기존 수정·중원·분당 3중심 체계를 폐기하고,
성남 전체를 **북부/중부/남부 생활권**으로 재편하는 작업을 진행 중이다.
즉, 지금이 생활권 경계 설정에 개입할 수 있는 시점이다.

### 문제 : 이 계획에는 두 가지 결함이 있다

**결함 ① 계획의 근거 데이터가 빈약하다**
도시개발계획 자료집의 데이터 분석은 단순 현황 나열 수준에 머물러 있다.
"수정구 고령인구 대비 복지시설이 분당구보다 얼마나 부족한가"와 같은
**수요-공급 교차 분석이 없다.**

**결함 ② 행정이 그은 생활권 경계가 시민의 실제 생활과 일치하는지 검증이 없다**
"북부 생활권으로 묶으면 된다"고 계획했지만,
북부 시민들이 실제로는 분당·판교로 통근·소비하고 있다면 그 계획은 현실과 동떨어진 것이다.
**시민이 체감하는 생활권과 행정이 설계한 생활권이 같다는 전제 자체가 검증된 적이 없다.**

### 목표 : 생산가능인구 기준으로 생활권을 재정의한다

분석의 핵심 질문을 하나로 고정한다.

> **"성남시 행정동별로 생산가능인구의 통근 접근성 격차가 실제로 존재하는가?
> 그리고 그 격차가 현재 생활권 개편 방향으로 해소되는가?"**

```mermaid
flowchart TD
    BG["배경\n성남시 2035 도시기본계획\n생활권 3중심 → 북부/중부/남부 재편 진행 중"]

    FLAW1["결함 ①\n계획 근거 데이터 빈약\n단순 현황 나열 수준\n수요-공급 교차분석 없음"]
    FLAW2["결함 ②\n체감 생활권 검증 없음\n생산가능인구의 실제 이동 경계 ≠\n행정 생활권 경계"]

    Q1["Q1. 격차가\n실제로 존재하는가?"]
    Q2["Q2. 어디가\n진짜 문제인가?"]
    Q3["Q3. 생활권을\n어떻게 재정의해야 하는가?"]

    A1["행정동별 통근 접근성 격차를\n주거·교통·고용 데이터로 정량화"]
    A2["주거지-일자리 미스매치 +\n교통 연결 불량이 겹치는 지역 특정"]
    A3["생산가능인구 이동 패턴 기반\n체감 생활권 경계 역산 및 제안"]

    CLAIM["메인 주장\n현재 생활권 개편 방향은 optimal하지 않다\n→ 생산가능인구 기준으로 재정의한 생활권 경계를 제안한다"]

    BG --> FLAW1
    BG --> FLAW2
    FLAW1 --> Q1
    FLAW2 --> Q2
    FLAW2 --> Q3
    Q1 --> A1
    Q2 --> A2
    Q3 --> A3
    A1 --> CLAIM
    A2 --> CLAIM
    A3 --> CLAIM
```

## 수요-공급 프레임

기존의 복지 수요-공급 프레임을 **생산가능인구 통근** 맥락으로 재정의한다.

| 구분 | 정의 | 데이터 |
|------|------|--------|
| **수요** | 생산가능인구(15~64세)가 거주하는 곳 | 행정동별 인구통계 + 주거 데이터 |
| **공급** | 일자리가 있는 곳 | 사업체 수 + 종사자 수 + 산업군 분포 |
| **연결** | 수요(주거지)와 공급(직장)을 잇는 교통 | 버스·지하철 OD + 통근 이동 데이터 |

> 수요-공급 GAP = 생산가능인구 밀도 대비 인근 일자리 접근성이 부족한 정도
> 이 GAP이 행정동별로 얼마나 다른지가 핵심 논거가 된다.

## 데이터 수집 구조

네 개의 독립 테이블을 **행정동 코드** 기준으로 JOIN하는 구조.

```mermaid
flowchart LR
    subgraph D["수요 데이터\n(KOSIS / 행안부)"]
        D1["행정동별 생산가능인구\n(15~64세)"]
        D2["1인가구 비율\n연령대별 인구 구조"]
    end

    subgraph H["주거 데이터\n(국토부 실거래가 시스템)"]
        H1["세대수 + 주택 재고\n(아파트·단독·다세대)"]
        H2["매매·전월세\n실거래가 평당가격"]
    end

    subgraph E["고용 데이터\n(통계청 / 행안부)"]
        E1["사업체 수\n종사자 수 (행정동별)"]
        E2["산업군 분포\n(제조·서비스·IT 등)"]
    end

    subgraph T["교통·통근 데이터\n(TAGO / 통계청 / 빅데이터캠퍼스)"]
        T1["버스·지하철 OD\n(승하차 통계)"]
        T2["통계청 생활이동 OD\n(통근 목적 이동)"]
        T3["카드 관외소비\n(소비 생활권 파악)"]
    end

    JOIN[("행정동 코드\nJOIN KEY")]

    D --> JOIN
    H --> JOIN
    E --> JOIN
    T --> JOIN
```

## 분석 3단계

```mermaid
flowchart TD
    subgraph STEP1["1단계 : 주거-일자리 수요-공급 불균형 파악"]
        direction LR
        S1_IN["수요 데이터\n+ 주거 데이터\n+ 고용 데이터"]
        S1_CALC["행정동별 집계\n- 생산가능인구 대비 인근 일자리 수\n- 주거 평당가격 vs 인근 고용 밀도\n- 자족도 = 동내 일자리 / 거주 생산가능인구"]
        S1_OUT(["공급 GAP Score\n(행정동 단위)"])
        S1_IN --> S1_CALC --> S1_OUT
    end

    subgraph STEP2["2단계 : 통근 접근성 분석"]
        direction LR
        S2_IN["교통 데이터\n+ 고용 데이터"]
        S2_CALC["도달 가능성 분석\n- 주요 고용지(판교·분당)까지 대중교통 소요시간\n- 환승 횟수\n- inter 생활권 통근 편의성\n  vs intra 생활권 통근 편의성"]
        S2_OUT(["접근성 Score\n(행정동 단위)"])
        S2_IN --> S2_CALC --> S2_OUT
    end

    subgraph STEP3["3단계 : 체감 생활권 도출"]
        direction LR
        S3_IN["통근 이동 OD\n+ 카드 관외소비"]
        S3_CALC["실제 이동 패턴과 행정 경계 비교\n- 통근 목적 이동의 관외 비율 (동별)\n- 주요 이동 목적지 Top 3\n- 체감 생활권 경계 역산"]
        S3_OUT(["체감이동 GAP Score\n(행정동 단위)"])
        S3_IN --> S3_CALC --> S3_OUT
    end

    STEP1 --> STEP2 --> STEP3
```

## 통합 Score 산출

```mermaid
flowchart TD
    G1(["공급 GAP Score\n주거-일자리 미스매치"])
    G2(["접근성 Score\n통근 접근성 불량 정도"])
    G3(["체감이동 GAP Score\n실제 이동 vs 행정 경계 괴리"])

    FORMULA["통합 Score 산출\n\nScore = w₁ × 공급GAP + w₂ × 접근성GAP + w₃ × 체감이동GAP\n\n가중치 결정 방법\n① PCA로 주성분 기반 자동 추출 (데이터 기반)\n② 선행연구 기반 전문가 설정 (설명 용이)"]

    RESULT(["행정동별 종합 취약 Score"])

    G1 --> FORMULA
    G2 --> FORMULA
    G3 --> FORMULA
    FORMULA --> RESULT
```

> **가중치 권고** : PCA로 1차 산출 후 선행연구와 비교·보정하는 방식 권장.
> 공모전 심사 설명을 위해 가중치 근거 문서화 필수.

## 시각화 플랜

각 시각화가 답해야 하는 질문을 기준으로 설계.

```mermaid
flowchart LR
    SCORE(["종합 Score\n(행정동 단위)"])

    subgraph VIZ["시각화 4종"]
        V1["① 단변량 분포\n히스토그램 / Box plot\n\n답할 질문\n- 생산가능인구는 어디 몰려있나?\n- 평당가격 분포의 Skewness는?\n- 일자리 밀도 분포는?"]
        V2["② 공간 분포\nChoropleth 지도 (다변량)\n\n답할 질문\n- 주거 밀집 vs 일자리 밀집 지역이\n  공간적으로 얼마나 어긋나는가?\n- 통근 접근성 취약 지역은 어디인가?"]
        V3["③ 변수 관계\nScatter plot\nCorrelation Heatmap\n\n답할 질문\n- 평당가격과 통근 접근성은\n  어떤 관계인가?\n- 일자리 밀도와 이동 패턴의 관계?"]
        V4["④ 지역 유형화\nK-means Clustering\n\n답할 질문\n- 비슷한 통근 패턴의 동끼리 묶으면?\n- 그 유형이 행정 생활권 경계와\n  어떻게 어긋나는가?"]
    end

    SCORE --> V1
    SCORE --> V2
    SCORE --> V3
    SCORE --> V4
```

## 지역 유형 분류

클러스터링 결과를 아래 4개 유형으로 프레이밍.

```mermaid
flowchart TD
    CLUSTER["K-means Clustering 결과\n(주거·고용·교통·이동 변수 기반)"]

    TA["Type A : 자족형\n주거-일자리 균형\n통근 이동 짧음\n예상 : 판교·분당 일부"]
    TB["Type B : 직주분리형\n주거↑ 일자리↓\n타 생활권으로 통근 유출\n예상 : 위례·수정구 일부"]
    TC["Type C : 접근성 불량형\n일자리는 인근에 있으나\n교통 연결이 안 됨\n예상 : 중원구 구시가지"]
    TD["Type D : 복합 취약형\n일자리↓ 접근성↓ 통근유출↑\n세 문제 동시 발생\n예상 : 수정구 북부"]

    CLUSTER --> TA
    CLUSTER --> TB
    CLUSTER --> TC
    CLUSTER --> TD
```

| 유형 | 공급 GAP | 접근성 | 체감이동 GAP | 의미 | 처방 | 우선순위 |
|------|----------|--------|--------------|------|------|----------|
| **A** 자족형 | 낮음 ✅ | 양호 ✅ | 낮음 ✅ | 주거-일자리 균형, 자급자족 | 현상 유지 | 낮음 |
| **B** 직주분리형 | 높음 ❌ | 보통 🔶 | 높음 ❌ | 주거만 있고 일자리 부족, 통근 유출 | 고용 거점 확충 | 중간 |
| **C** 접근성 불량형 | 낮음 ✅ | 불량 ❌ | 보통 🔶 | 일자리는 인근에 있지만 교통 연결 안됨 | 교통 연결 보완 | 중간 |
| **D** 복합 취약형 | 높음 ❌ | 불량 ❌ | 높음 ❌ | 일자리·교통·이동 세 문제 동시 발생 | 고용+교통+경계 재조정 | **최우선** |
